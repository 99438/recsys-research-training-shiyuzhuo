# 第六课精读论文选择与核心问题

## 精读对象

**Self-Attentive Sequential Recommendation (SASRec)**
Wang-Cheng Kang, Julian J. McAuley — ICDM 2018（CCF-B）
DOI: 10.1109/ICDM.2018.00035

---

## 为什么选这篇（4 点理由）

1. **领域地基**——本次调研的 11 篇候选论文中，8 篇（BERT4Rec、CL4SRec、ICLRec、DCRec、FMLP-Rec、Recformer、Temporal Graph CL、以及三篇综述）都把 SASRec 作为必比 baseline 或作为方法讨论的起点。读透 SASRec 等于同时打开了 8 篇后续工作的大门。

2. **方法典型且清晰**——self-attention 在序列推荐里的推导链在这篇论文里最完整、变量最少、公式最干净。相比 BERT4Rec 加了双向 mask、CL4SRec 加了对比分支，SASRec 是"最小完备形态"，适合作为理解 Transformer-based 推荐的起点。

3. **可完整复现**——官方 TensorFlow 代码开源（github.com/kang205/SASRec），社区 PyTorch 复现完善；主实验数据集 Amazon Beauty 仅 40MB 左右，可以在我本地笔记本上完整跑通训练与评估，为精读补上"我亲手跑过"的深度。

4. **难度匹配我的当前水平**——不需要 A100 GPU（LLM4Rec）、不需要处理图结构（DCRec/TGCL4SR）、不需要复杂的多模态数据（Recformer），学习曲线平滑，能在第六课精读的时间预算内完全消化。

## 阅读该论文时最想弄懂的 3 个核心问题

### 问题 1（关于**方法**）：SASRec 里的 self-attention 为什么用**单向 causal mask**，而不像 BERT4Rec 那样用双向？

- **拆解方向**：
  - 单向 mask 与 next-item prediction 任务的因果结构一致——预测第 t+1 个物品时只能看到前 t 个
  - 但 BERT4Rec 用双向 + Cloze 任务也能达到甚至超过 SASRec，为什么？
- **想搞清**：因果 mask 是**任务约束还是建模假设**？如果我把 SASRec 改成双向 + next-item 目标会崩吗？
- **验证方式**：读论文 §III-B（Self-Attention Block）与 §V-B（对比 BERT4Rec 的分析）；有时间做一次消融，把 mask 换掉看效果。

### 问题 2（关于**实验**）：论文在 Beauty / Games / Steam / ML-1M 四个数据集上取得 SOTA，但**评估协议**（负采样策略 / leave-one-out / 全量排序）是怎么选的？这个选择会不会影响结论？

- **拆解方向**：
  - 论文用的是 leave-one-out + 采样 100 个负例的评估协议
  - 后续研究（如 Rendle 2020 "On Sampled Metrics for Item Recommendation" 一文）指出**采样式评估**可能高估模型效果
  - CL4SRec、BERT4Rec 用的是同一套协议，但也有工作改用全量排序
- **想搞清**：SASRec 的相对提升在**全量排序协议**下是否仍然显著？若不显著，说明什么？
- **验证方式**：读论文 §IV-A（Evaluation Metrics）；对照 Boka 2024 综述中"评估协议"章节的讨论。

### 问题 3（关于**局限**）：论文自己承认哪些局限？我如果把 SASRec 作为下一步研究的起点，最有价值的**扩展方向**是什么？

- **拆解方向**：
  - 从论文 §VI（Conclusion）与后续工作反向看：
    - **数据稀疏** → 已被 CL4SRec / ICLRec / DCRec 用对比学习补上
    - **单向 mask 局限** → 已被 BERT4Rec 补上
    - **纯 ID 建模、无内容特征** → 已被 Recformer 用文本表示补上
    - **未处理噪声** → 已被 FMLP-Rec 用频域滤波补上
- **想搞清**：哪些方向**尚未被充分解决**？哪个方向最适合作为我未来的研究切入点？
- **验证方式**：结合 SASRec 结论 + Pan 2026 综述"未来方向"章节 + Chen 2026"多行为"综述交叉判断。
