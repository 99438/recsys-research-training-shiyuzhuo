### 深度学习环境的核心工具

#### Conda环境管理
- **作用**：为每个项目创建独立的Python环境，避免依赖冲突
- **类比**：给每个项目准备一个独立的工具箱
- **关键理解**：不同项目往往需要不同版本的Python和包，隔离环境是必须的

```powershell
# 创建独立环境
conda create -n lesson03 python=3.10 -y

# 激活环境
conda activate lesson03

# 查看所有环境
conda env list

# 删除环境
conda remove -n env_name --all
```

#### PyTorch版本选择
- **CPU版**：只有PyTorch核心代码，约123MB，速度慢
- **GPU版**：包含CUDA运行时库、cuDNN、cuBLAS，约2.4GB，速度快10-100倍
- **关键理解**：同一环境中只能装一个版本，torch包名一样会冲突

```powershell
# CPU版
python -m pip install torch==2.12.1 --index-url https://download.pytorch.org/whl/cpu

# GPU版（需匹配驱动CUDA版本）
python -m pip install torch==2.5.1 --index-url https://download.pytorch.org/whl/cu121
```

### GPU与CUDA基础

#### GPU检查命令
```bash
nvidia-smi
```

**关键输出解读**：
- `Driver Version`：NVIDIA驱动版本
- `CUDA Version`：驱动**支持上限**的CUDA版本
- `Memory-Usage`：显存使用情况
- `GPU-Util`：GPU利用率
- `Processes`：正在使用GPU的进程

#### CUDA版本关系（很容易混淆）
```
NVIDIA驱动 → 支持CUDA上限 → 决定PyTorch可安装的CUDA版本
```

- **规则**：PyTorch的CUDA版本 ≤ 驱动支持的CUDA版本
- **例子**：驱动支持CUDA 12.3 → 可以装cu121, cu118，不能装cu126

#### 验证PyTorch的GPU支持
```python
import torch
print(torch.__version__)              # PyTorch版本
print(torch.version.cuda)             # PyTorch内嵌的CUDA运行时版本
print(torch.cuda.is_available())      # GPU是否可用
print(torch.cuda.get_device_name(0))  # GPU型号
```

**重要理解**：
- `nvidia-smi`能看到GPU ≠ PyTorch能用GPU
- 需要同时满足：驱动正常 + PyTorch是GPU版本 + CUDA版本兼容

## 重点概念理解

### 1. 环境配置的三层结构
```
硬件（NVIDIA GPU）
    ↓
驱动（NVIDIA Driver + CUDA支持）
    ↓
框架（PyTorch + CUDA运行时）
    ↓
应用（我的模型代码）
```

任何一层出问题，上层都会失败。

### 2. 环境隔离的必要性
- **不好的做法**：所有包都装在base环境
- **好的做法**：每个项目一个独立环境
- **命名规范**：环境名清晰体现用途（如 `lesson03`, `sasrec_gpu`）

### 3. 版本管理原则
- **课程统一版本**：确保所有人环境一致，便于排查问题
- **实际情况**：本地驱动限制可能需要调整版本
- **应对方法**：说明差异原因，验证代码兼容性

### 4. 敏感信息的边界
| 位置 | 可以放 | 不能放 |
|------|--------|--------|
| Git仓库 | 代码、文档 | 密码、Token、密钥 |
| NAS公共目录 | 共享数据 | 个人凭证 |
| 代码 | 逻辑 | 硬编码的Token |
| 截图 | 命令示例 | 含Token的输出 |

## 遇到的关键问题

### 问题1：PyTorch 2.12.1不支持我的CUDA版本

**现象**：
```
ERROR: Could not find a version that satisfies the requirement torch==2.12.1
(from versions: 2.1.0+cu121, ..., 2.5.1+cu121)
```

**排查过程**：
1. 运行 `nvidia-smi` → 驱动支持CUDA 12.3
2. 课程推荐版本 `torch==2.12.1+cu126` → cu126需要CUDA 12.6驱动
3. 查看cu121可用范围 → 最高PyTorch 2.5.1
4. 选择 `torch==2.5.1+cu121`（兼容CUDA 12.3驱动）

**解决**：
```powershell
python -m pip install torch==2.5.1 --index-url https://download.pytorch.org/whl/cu121
```

**学到的教训**：不能盲目照抄课程命令，需要根据本地环境调整

### 问题2：Conda activate 在 PowerShell 中不生效

**现象**：`conda activate lesson03` 后，`python --version` 仍显示 base 环境

**原因**：PowerShell 中 conda 未正确初始化

**解决方法**：使用完整路径直接调用
```powershell
& "D:\anaconda\envs\lesson03\python.exe" [命令]
```

**深层理解**：`conda activate` 本质是修改 PATH 环境变量，如果没有初始化就无法正确修改

### 问题3：CPU版 vs GPU版切换

**现象**：同一环境已装CPU版，想改GPU版

**错误做法**：直接 `pip install` GPU版（会冲突）

**正确做法**：
```powershell
# 方案A：卸载后重装
pip uninstall torch -y
pip install torch --index-url https://download.pytorch.org/whl/cu121

# 方案B（更好）：新建独立环境
conda create -n lesson03-gpu python=3.10 -y
```
