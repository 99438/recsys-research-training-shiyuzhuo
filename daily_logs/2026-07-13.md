#### Python容器类型复习
- **list**: 有序可修改，适合动态数据
- **tuple**: 有序不可修改，适合固定结构（如shape）
- **set**: 无序不重复，适合去重和集合运算
- **dict**: 键值映射，适合结构化数据
- **关键理解**: 容器选择取决于使用场景，不是随意选择

#### Tensor基础
```python
# 核心属性
tensor.shape          # 数据结构
tensor.ndim           # 维度数量
tensor.dtype          # 数据类型（float32/long）
tensor.device         # 计算设备（cpu/cuda）
tensor.requires_grad  # 是否跟踪梯度
```

**重点理解**: Tensor不是"另一种列表"，而是专为深度学习设计的数值计算对象，支持GPU加速和自动求导。

#### 机器学习核心概念

**参数 vs 超参数**
- 参数: 从数据中学习（权重、偏置、Embedding）
- 超参数: 人工设定（学习率、batch size、epoch）

**数据划分**
- 训练集: 学习参数
- 验证集: 调参和选择最佳模型
- 测试集: 最终评估（只看一次）

**训练循环三步骤**
```python
optimizer.zero_grad()  # 1. 清除旧梯度（梯度会累积）
loss.backward()        # 2. 计算新梯度（自动求导）
optimizer.step()       # 3. 更新参数
```

- 线性模型: 特征加权求和
- MLP: 多层非线性变换
- CNN: 局部窗口提取模式
- RNN/LSTM: 序列状态传递
- Transformer: Self-Attention机制

### 完整二维分类实验

#### DataLoader原理
```python
# Dataset: 保存样本，支持索引
train_set = TensorDataset(train_x, train_y)

# DataLoader: 自动组batch、打乱、迭代
train_loader = DataLoader(train_set, batch_size=16, shuffle=True)

# 使用
for batch_x, batch_y in train_loader:
    # 训练代码
```

**关键理解**: DataLoader只负责数据组织，不计算Loss也不更新参数。

#### 完整实验流程

**数据流shape变化**
```
全部数据:  [240, 2]   → 240个样本，每个2维特征
一个batch: [16, 2]    → 16个样本的batch
模型输出:  [16, 1]    → Linear输出
squeeze:   [16]       → 去掉维度1，与标签对齐
标签:      [16]       → batch_labels
Loss:      []         → 必须是标量
```

## 重点概念理解

### 1. 五大核心概念
- **Tensor**: 多维数值数组，深度学习的基础数据结构
- **Batch**: 一次训练使用的样本子集（如16条）
- **Epoch**: 完整遍历训练集一次
- **Logit**: 模型输出的原始分数（未经sigmoid）
- **Loss**: 衡量预测与真实值的差距，必须是标量

### 2. 训练 vs 评估
| 阶段 | 模式 | 梯度 | 参数更新 |
|------|------|------|----------|
| 训练 | `model.train()` | ✓ | ✓ |
| 评估 | `model.eval()` + `torch.no_grad()` | ✗ | ✗ |

### 3. Shape追踪很重要
训练过程中最常见的错误就是shape不匹配，必须确保：
- logits和labels形状一致（都是`[B]`或都是`[B, 1]`）
- Loss是标量（`[]`），不是向量

## 遇到的关键问题

### 问题1: 为什么需要squeeze？
**现象**: `nn.Linear(2, 1)`输出是`[B, 1]`，但标签是`[B]`，导致Loss计算报错

**解决**: 
```python
logits = model(x).squeeze(1)  # [B, 1] → [B]
```

### 问题2: 为什么要保存state_dict而不是整个模型？
**理解**: 
- `state_dict()`只保存参数，节省内存
- 需要用`deepcopy`防止后续训练修改参数
- 恢复时用`model.load_state_dict(best_state)`

### 问题3: 验证集和测试集的区别
- **验证集**: 训练中反复查看，用于选模型和调参
- **测试集**: 最后只看一次，评估真实性能
- **目的**: 避免在验证集上"过拟合"
