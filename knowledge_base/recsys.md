# 推荐系统笔记

## 基础概念

### 核心要素

- **用户（User）**：推荐系统的服务对象
- **物品（Item）**：被推荐的对象（电影、商品、文章等）
- **交互数据（Interaction）**：用户与物品的行为记录（点击、购买、评分等）

### 推荐任务类型

- **点击率预测（CTR）**：预测用户点击某物品的概率
- **Top-K 推荐**：为用户推荐 K 个最可能感兴趣的物品
- **序列推荐**：基于用户历史行为序列预测下一个物品
- **召回（Recall）**：从海量物品中快速筛选候选集
- **排序（Ranking）**：对候选物品进行精细排序

### 重要术语

**数据相关**

- **显式反馈（Explicit Feedback）**：用户明确表达的偏好（评分、点赞）
- **隐式反馈（Implicit Feedback）**：从行为推断的偏好（点击、浏览）
- **负采样（Negative Sampling）**：构造负样本用于模型训练

**模型相关**

- **Embedding**：将用户/物品映射到低维向量空间
- **协同过滤（Collaborative Filtering）**：基于用户-物品交互矩阵的推荐
- **内容推荐（Content-based）**：基于物品特征的推荐

**评价指标**

- **AUC**：分类模型的评价指标
- **Recall@K**：推荐列表中相关物品的召回率
- **NDCG**：考虑排序位置的评价指标
- **HR（Hit Rate）**：命中率

## 推荐系统流程

1. **数据收集**：用户行为、物品特征、上下文信息
2. **特征工程**：构造用户特征、物品特征、交互特征
3. **模型训练**：选择合适的推荐算法并训练
4. **离线评估**：在测试集上评估模型效果
5. **在线部署**：A/B 测试验证实际效果

## 经典模型

### 1. 流行度推荐（Popularity-based）

- **推荐依据**：全局交互统计（点击、购买、播放、评分）
- **适用场景**：新用户、缺少个性化数据、实验简单基线
- **优势**：简单、快速、无需个人历史
- **问题**：缺少个性化、强化热门偏置、压缩长尾物品曝光

### 2. 内容推荐（Content-based）

- **推荐依据**：物品内容特征相似度（类别、标签、文本、图片）
- **核心思路**：根据用户过去喜欢的物品建立兴趣表示，推荐内容特征相似的其他物品
- **适用场景**：有物品特征、新物品上架
- **优势**：推荐依据直观、可利用新物品内容信息
- **问题**：依赖物品特征质量、容易重复推荐相似内容、兴趣收窄

### 3. 协同过滤（Collaborative Filtering）

#### UserCF（基于用户的协同过滤）

- **推荐依据**：找到与目标用户兴趣相似的用户，汇总邻居喜欢的物品
- **推荐过程**：用户-物品交互 → 相似用户选择 → 邻居行为聚合 → 候选物品推荐
- **优势**：利用群体协同信号、方法直觉清楚
- **问题**：交互稀疏、用户兴趣变化、用户冷启动、大规模近邻计算成本

#### ItemCF（基于物品的协同过滤）

- **推荐依据**：根据共同交互学习物品相似关系，推荐与用户过去喜欢物品相似的其他物品
- **推荐过程**：用户-物品交互 → 物品相似关系 → 用户历史物品 → 候选物品推荐
- **优势**：物品关系较稳定、容易形成"因为喜欢A，所以推荐B"的解释
- **问题**：新物品缺少交互、热门物品共现偏置、推荐结果可能过窄

### 4. 矩阵分解（Matrix Factorization）

- **推荐依据**：将用户和物品映射到同一个低维隐空间，用户向量与物品向量的点积预测偏好
- **使用信息**：用户-物品评分矩阵或隐式反馈矩阵
- **数学形式**：奇异值分解 $A = U\Sigma V^T$
- **优势**：能捕捉潜在语义、可扩展到大规模
- **问题**：用户和物品冷启动、隐向量解释有限、表示需要训练学习

## 现代前沿方法（2022-2026）

### 研究对象分类

- **序列推荐**：行为顺序和兴趣变化
- **图推荐**：用户、物品和关系结构
- **多模态推荐**：文本、图片、音频和视频

### 方法机制分类

#### 1. 对比学习推荐

- **核心思想**：使用自监督信号改善表示学习
- **常见任务**：序列表征、推荐排序
- **主要问题**：正样本或增强策略可能偏离真实偏好

#### 2. 频域推荐（Frequency Domain Recommendation）

- **核心思想**：将用户行为序列从时域转换到频域，学习或过滤不同频率成分
- **代表方法**：
  - **FMLP-Rec** (WWW 2022)：可学习频域过滤器与全MLP结构
  - **FEARec** (SIGIR 2023)：联合建模时域依赖、频率成分和周期行为
- **技术路线**：
  1. 将用户行为序列转换到频域（傅里叶变换）
  2. 通过可学习过滤器选择有效频率成分
  3. 转换回时域进行下一物品预测
- **适用任务**：序列推荐、下一物品预测
- **优势**：能够自动学习任务相关的频率模式、过滤噪声、捕捉周期性
- **主要问题**：频率成分的推荐语义不直观、数据中不一定存在稳定周期、计算开销

#### 3. 扩散推荐（Diffusion-based Recommendation）

- **核心思想**：对交互或目标物品表示逐步加噪，学习反向去噪过程，恢复用户偏好
- **代表方法**：
  - **DiffRec** (SIGIR 2023)：对用户-物品交互向量进行去噪
  - **DiffuRec** (ACM TOIS 2024)：生成序列中的目标物品表示
- **技术路线**：
  1. 在训练时对交互向量逐步加入高斯噪声
  2. 学习反向去噪网络
  3. 在推理时从噪声中恢复用户偏好，生成推荐列表
- **适用任务**：Top-N推荐、交互去噪、下一物品预测
- **优势**：可以处理稀疏和噪声数据、提供生成式建模能力
- **主要问题**：多步去噪增加训练和推理成本、对噪声过程/步数/实验设置较敏感

#### 4. 生成式检索（Generative Retrieval）

- **传统召回方式**：学习用户/查询向量 → 近似最近邻搜索 → 候选物品
- **生成式检索思路**：为物品构造具有语义的离散ID，根据历史物品ID直接生成目标物品ID
- **代表方法**：
  - **TIGER** (NeurIPS 2023)：使用语义ID进行生成式推荐检索
- **技术路线**：
  1. 为物品构造语义ID（通过聚类、层次化编码等方式）
  2. 根据用户历史物品的语义ID序列
  3. 自回归生成目标物品的语义ID
- **适用任务**：候选召回、下一物品预测
- **优势**：端到端学习、无需显式向量检索
- **主要问题**：语义ID和码本设计、物品目录更新、解码错误和推理效率

#### 5. 语言模型推荐

- **核心思想**：将用户、物品、交互、评论和任务指令转换为自然语言序列，使用统一的语言建模目标
- **代表方法**：
  - **P5** (RecSys 2022)：预训练、个性化提示和预测（Pretrain, Personalized Prompt & Predict）
- **输入形式**：用户信息、物品信息、交互与评论、任务指令（统一文本接口）
- **输出形式**：推荐列表、评分预测、推荐解释、评论生成
- **适用任务**：推荐列表生成、评分预测、推荐解释和评论生成
- **主要问题**：物品ID与语言语义对齐、生成结果约束、不同任务的统一评价

#### 6. 大模型推荐（LLM-enhanced Recommendation）

**三种定位**：

1. **作为推荐器或生成器**
   - 直接生成推荐、解释或对话结果

2. **作为对齐后的推荐模型**
   - 使用推荐数据微调通用大模型
   - **TALLRec** (RecSys 2023)：用于说明推荐数据对齐

3. **作为知识与特征增强器**
   - 补充物品属性、用户画像和关系信息
   - **LLMRec** (WSDM 2024)：用于说明图和特征增强

**主要价值**：语言理解、外部知识、跨任务表达和自然语言交互

**主要问题**：成本、时延和幻觉、物品落地约束、隐私和评价困难

**重要理解**：大模型增强的图推荐不是由大模型直接输出列表，而是用大模型增强图的特征表示

## 多阶段推荐系统架构

### 完整链路

```
海量物品 → 召回(百万→千) → 排序 (千→百) → 重排(百→十) → 推荐列表
```

### 各阶段特点

#### 召回（Recall）

- **目标**：从海量物品中快速筛选候选集
- **规模**：百万级 → 数百或数千
- **可用方法**：流行度、协同过滤、向量检索、序列模型、生成式检索
- **要求**：速度快、覆盖面广

#### 排序（Ranking）

- **目标**：精细预测候选相关性
- **规模**：数千 → 数十或数百
- **技术**：综合用户、物品、交互和上下文特征
- **要求**：预测精准、特征丰富

#### 重排（Re-ranking）

- **目标**：加入多样性、规则和安全约束
- **规模**：数十 → 最终展示列表
- **考虑因素**：多样性、新颖性、业务规则、安全和重复控制
- **要求**：用户体验、业务目标、合规要求

### 研究模型 vs 真实系统

| 维度   | 论文研究           | 真实系统                   |
| ------ | ------------------ | -------------------------- |
| 复杂度 | 一个任务、一个模型 | 多路召回、多个模型         |
| 约束   | 主要关注指标       | 规则、安全、内容治理       |
| 数据   | 离线静态数据       | 实时数据、延迟要求、稳定性 |
| 评价   | 离线指标           | 线上A/B测试、用户体验      |

**重要认知**：

- 离线指标提高 ≠ 用户体验提高
- 单一模型结果 ≠ 完整系统效果
- 阅读论文时需要判断研究对象和系统位置

## 推荐系统合规

### 合规基础

- 在具有合法处理依据和明确目的的前提下使用必要数据
- 清晰告知个性化推荐的基本情况
- 提供不针对个人特征的选项或便捷关闭方式
- 允许用户选择、修改或删除用于推荐的个人标签

### 高风险或禁止做法

- 过度收集、暗中追踪、超出原目的使用或非法共享个人信息
- 根据用户画像实行不合理差别定价或其他差别交易条件
- 诱导沉迷、过度消费，或者向未成年人推送有害内容
- 操纵榜单、热搜和检索结果，制造虚假点赞、评论或账号
- 推荐违法信息，或者利用算法规避监管、实施不正当竞争

### 上线检查清单

- [ ] 自动化决策是否透明、公平、公正
- [ ] 用户是否能够拒绝、关闭、申诉并要求必要说明
- [ ] 是否处理敏感个人信息或未成年人个人信息
- [ ] 是否完成必要的影响评估、安全管理、备案或安全评估

- 《个人信息保护法》

## 推荐实验范式

### 完整实验流程

```
明确任务 → 选择数据 → 数据划分 → 构造样本 → 
选择基线 → 训练模型 → 统一评价 → 公平比较 → 边界结论
```

**核心原则**：模型只是实验链条中的一个环节，数据、样本和比较条件同样影响结论可信度

### 1. 任务、数据与划分

#### 任务定义

- 评分预测、Top-N推荐或下一物品预测
- 明确预测目标和评价标准

#### 数据说明

- 用户、物品和交互类型
- 时间信息、数据规模和稀疏程度
- 过滤和预处理规则

#### 数据划分

- **训练集**：学习模型参数
- **验证集**：模型选择和参数调整
- **测试集**：最终评价（只看一次）

**比较要求**：使用相同的数据版本、预处理和划分

### 2. 时间信息泄漏（最常见的实验错误）

#### 错误做法

- 随机打散全部序列行为
- 未来点击进入训练数据
- 使用未来统计量预测过去行为

#### 正确做法

- **按时间顺序划分数据**
- 使用过去行为训练
- 使用较新的行为验证
- 使用最后的行为测试

#### 其他泄漏来源

- 物品热度和全局统计（必须只使用训练集时间点之前的统计）
- 特征构造（不能使用未来信息）
- 负样本池（不能包含未来出现的物品）

**基本原则**：数据划分必须与真实预测场景一致

### 3. 负采样与候选集

**重要区分**：二者都可能从未交互物品中取样，但不能混为一谈

| 比较维度 | 训练负采样         | 评价候选集                     |
| -------- | ------------------ | ------------------------------ |
| 目的     | 构造模型学习样本   | 决定测试时从哪些物品中寻找目标 |
| 环节     | 模型训练           | 模型评价                       |
| 主要影响 | 模型学到的区分边界 | 指标难度和结果可比性           |

**记录要求**：

- 负样本数量和采样分布需要记录
- 全量评价和采样评价不能直接比较
- 候选集不同，同一个Recall@K也不能直接比较

### 4. 基线与公平比较

#### 基线作用

回答新方法究竟比什么更好

#### 基线层次

1. **简单基线**：流行度推荐
2. **经典方法**：ItemCF
3. **传统深度方法**：矩阵分解
4. **现代方法**：合适的现代推荐模型（如SASRec、LightGCN等）

#### 一致条件要求

- 数据版本、预处理和数据划分
- 评价候选集和负采样口径
- 推荐列表长度K
- 训练预算、调参条件和指标定义

#### 不公平做法

- 只选择明显较弱的基线
- 使用不合理的基线实现或参数
- 在不同实验条件下比较指标
- 给新方法更多的训练epoch或调参机会

### 5. 评价指标

#### 准确性指标

| 指标       | 主要问题                     | 适用场景 |
| ---------- | ---------------------------- | -------- |
| Recall@K   | 相关物品中有多少被找回       | 召回阶段 |
| HitRate@K  | 推荐列表是否至少命中一个目标 | 整体评价 |
| NDCG@K     | 命中物品是否位于列表前部     | 排序阶段 |
| RMSE / MAE | 预测评分与真实评分相差多少   | 评分预测 |
| AUC        | 分类模型的评价指标           | CTR预测  |

#### 准确性之外的维度

- **覆盖率（Coverage）**：被推荐的物品占总物品的比例
- **多样性（Diversity）**：推荐列表中物品的差异程度
- **新颖性（Novelty）**：推荐不太热门但用户可能感兴趣的物品
- **用户满意度**：需要线上A/B测试
- **效率**：计算时间、内存占用
- **鲁棒性**：对攻击、噪声的抵抗能力

**重要**：指标必须与任务和评价候选集相对应

### 6. 实验结论边界

#### 可以说明

- 模型在当前数据、任务、指标和设置下表现更好

#### 不能自动推广

- 一个数据集的结果代表所有应用领域
- 准确性提高代表多样性和效率一定提高
- 离线指标提高代表线上体验一定提高

#### 结论要求

- 写清数据、任务、基线和实验条件
- 说明结果的适用范围和限制
- 承认方法的局限性

### 7. 公平比较检查清单

实验能否直接比较的判断标准：

- [ ] 数据版本和预处理方式相同
- [ ] 数据划分方式相同（训练/验证/测试比例、时间划分点）
- [ ] 训练负采样策略相同（数量、分布）
- [ ] 评价候选集相同（全量 vs 采样、采样比例）
- [ ] 推荐列表长度K相同
- [ ] 训练预算相当（epoch数、调参次数）
- [ ] 评价指标定义相同
- [ ] 无时间信息泄漏

**如果以上任何一项不同，结果不能直接比较**

```python
from torch.utils.data import TensorDataset, DataLoader

train_set = TensorDataset(train_features, train_labels)
train_loader = DataLoader(
    train_set,
    batch_size=256,
    shuffle=True,  # 训练时打乱
    num_workers=0

```

#### 3. 定义模型

```python
import torch.nn as nn

class RecommenderModel(nn.Module):
    def __init__(self, num_users, num_items, embedding_dim):
        super().__init__()
        self.user_embedding = nn.Embedding(num_users, embedding_dim)
        self.item_embedding = nn.Embedding(num_items, embedding_dim)
    
    def forward(self, user_ids, item_ids):
        user_vectors = self.user_embedding(user_ids)
        item_vectors = self.item_embedding(item_ids)
        scores = (user_vectors * item_vectors).sum(dim=1)
        return scores
```

#### 4. 训练循环

```python
model = RecommenderModel(num_users, num_items, embedding_dim=64)
criterion = nn.BCEWithLogitsLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)

for epoch in range(num_epochs):
    # 训练阶段
    model.train()
    for batch in train_loader:
        user_ids, item_ids, labels = batch
        
        # 前向传播
        logits = model(user_ids, item_ids)
        loss = criterion(logits, labels)
        
        # 反向传播与参数更新
        optimizer.zero_grad()  # 1. 清除旧梯度
        loss.backward()        # 2. 计算新梯度
        optimizer.step()       # 3. 更新参数
    
    # 验证阶段
    model.eval()
    with torch.no_grad():
        valid_loss = evaluate(model, valid_loader)
        
    # 保存最佳模型
    if valid_loss < best_valid_loss:
        best_state = model.state_dict()
```

#### 5. 关键概念

- **Batch**：一次训练使用的样本子集（如256条）
- **Epoch**：完整遍历训练集一次
- **Logit**：模型输出的原始分数（未经sigmoid）
- **Loss**：衡量预测与真实标签的差距（必须是标量）
- **梯度**：Loss对每个参数的偏导数，指示参数调整方向

#### 6. 数据流中的Shape变化

```
全部数据:  [N, F]         N个样本，F个特征
一个batch: [B, F]         B个样本的batch
模型输出:  [B, 1]         每个样本一个logit
squeeze:   [B]            去掉维度1
标签:      [B]            与logits对齐
Loss:      []             标量
```

#### 7. 训练vs评估

| 阶段      | 模式                               | 梯度计算 | 参数更新 | Dropout | BatchNorm         |
| --------- | ---------------------------------- | -------- | -------- | ------- | ----------------- |
| 训练      | `model.train()`                    | ✓        | ✓        | 启用    | 使用当前batch统计 |
| 验证/测试 | `model.eval()` + `torch.no_grad()` | ✗        | ✗        | 关闭    | 使用全局统计      |

#### 8. 如何判断训练正常

- Loss持续下降
- 验证集表现合理
- 无明显过拟合（验证loss不持续上升）
- 测试集结果与验证集接近
