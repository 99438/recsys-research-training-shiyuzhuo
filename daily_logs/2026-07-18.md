## 第4次课核心内容

### 课程四大模块

#### 模块1：推荐系统全景

**信息过载与推荐系统的价值**
- **搜索引擎**：需求明确，用户主动输入查询
- **推荐系统**：需求不明确，系统主动筛选候选
- **个性化价值**：帮助长尾内容与潜在用户匹配，降低选择成本

**推荐系统的四个分析维度**
1. **应用领域**：电商、内容、社交、广告、位置服务、知识科研
2. **任务形式**：评分预测、点击预测、Top-N推荐、下一物品预测
3. **方法路线**：流行度、内容、协同过滤、矩阵分解、深度学习、序列、图、多模态、大模型
4. **系统阶段**：召回、排序、重排

**重要认知**
- 同一个方法可以有多个标签（如"深度序列推荐"）
- 推荐方法可以服务于多个系统阶段
- 召回、排序、重排不是新的算法家族，而是系统架构的不同阶段

#### 模块2：推荐数据与经典方法

**推荐数据的核心要素**
- **用户**：人、账号、设备或匿名会话
- **物品**：商品、视频、音乐、论文、地点
- **交互**：浏览、点击、收藏、购买、评分
- **上下文**：时间、地点、设备、入口

**用户反馈类型**
- **显式反馈**：评分、喜欢/不喜欢、明确标签
- **隐式反馈**：曝光、点击、停留、收藏、购买
- **关键理解**：没有交互 ≠ 不喜欢（可能是没看到）

**负采样的两面性**
- 未交互物品可能是"不喜欢"，也可能是"没看到"
- 负采样是一种建模假设，会影响模型学习
- 训练负采样 ≠ 评价候选集（目的和影响不同）

**五大经典方法对比**

| 方法     | 推荐依据   | 适用场景           | 典型问题         |
|----------|------------|--------------------|------------------|
| 流行度   | 全局热度   | 新用户、简单基线   | 热门偏置         |
| 内容推荐 | 内容相似   | 有物品特征、新物品 | 兴趣收窄         |
| UserCF   | 相似用户   | 群体兴趣可迁移     | 稀疏与用户冷启动 |
| ItemCF   | 相似物品   | 物品关系较稳定     | 新物品冷启动     |
| 矩阵分解 | 隐向量匹配 | 交互规模较大       | 冷启动与解释有限 |

**课堂思考题**
- 电商搜索框：搜索（但可以个性化排序）
- 短视频信息流：推荐
- 用户没点击某论文：不能直接标记为负例（可能没看到）

#### 模块3：现代方法与真实系统

**近期研究方向地图**

基础建模工具：深度学习（用于召回、排序、序列、图、多模态）

研究对象：
- **序列推荐**：行为顺序和兴趣变化
- **图推荐**：用户、物品和关系结构
- **多模态推荐**：文本、图片、音频、视频

常见机制：
- **对比学习**：自监督信号改善表示
- **频域**：在频域中处理序列信号
- **扩散**：通过加噪和去噪建模偏好
- **生成式检索**：生成物品语义ID
- **大模型**：利用语言知识和自然语言接口

**五类现代方法详解**

1. **频域推荐**
   - **基本思路**：将行为序列从时域转换到频域，学习或过滤不同频率成分
   - **代表方法**：FMLP-Rec (WWW 2022)、FEARec (SIGIR 2023)
   - **适用任务**：序列推荐、下一物品预测
   - **主要问题**：频率成分的推荐语义不直观、数据中不一定存在稳定周期

2. **扩散推荐**
   - **基本思路**：对交互或目标物品表示逐步加噪，学习反向去噪过程
   - **代表方法**：DiffRec (SIGIR 2023)、DiffuRec (ACM TOIS 2024)
   - **适用任务**：Top-N推荐、下一物品预测
   - **主要问题**：多步去噪增加计算成本、对设置敏感

3. **生成式检索**
   - **传统召回**：学习用户/查询向量 → 近似最近邻搜索 → 候选物品
   - **生成式检索**：为物品构造语义ID → 根据历史物品ID生成目标物品ID
   - **代表方法**：TIGER (NeurIPS 2023)
   - **主要问题**：语义ID和码本设计、物品目录更新、解码错误

4. **语言模型推荐**
   - **基本思路**：将用户、物品、交互转换为自然语言序列，使用统一语言建模目标
   - **代表方法**：P5 (RecSys 2022)
   - **适用任务**：推荐列表生成、评分预测、推荐解释、评论生成
   - **主要问题**：物品ID与语言语义对齐、生成结果约束

5. **大模型推荐的三种定位**
   - **作为推荐器/生成器**：直接生成推荐、解释或对话结果
   - **作为对齐后的推荐模型**：用推荐数据微调通用大模型（TALLRec, RecSys 2023）
   - **作为知识与特征增强器**：补充物品属性、用户画像和关系信息（LLMRec, WSDM 2024）

**多阶段推荐系统架构**

```
海量物品 → 召回 → 排序 → 重排 → 推荐列表
         （百万→千）（千→百）（百→十）
```

- **召回**：快速筛选候选（流行度、协同过滤、向量检索、生成式检索）
- **排序**：精细预测相关性（综合用户、物品、交互、上下文特征）
- **重排**：多样性、新颖性、业务规则、安全和重复控制

**研究模型 vs 真实系统**
- 论文研究：一个任务、一个模型、一个系统阶段
- 真实系统：多路召回、多个模型、规则约束、实时数据、稳定性要求
- **重要认知**：离线指标提高 ≠ 用户体验提高

**推荐系统合规要点**
- 清晰告知个性化推荐情况
- 提供不针对个人特征的选项或便捷关闭方式
- 允许用户选择、修改或删除用于推荐的标签
- 禁止：过度收集、差别定价、诱导沉迷、操纵榜单、推荐违法信息

#### 模块4：推荐实验范式

**实验流程**
```
明确任务 → 选择数据 → 数据划分 → 构造样本 → 
选择基线 → 训练模型 → 统一评价 → 公平比较 → 边界结论
```

**核心原则：模型只是实验链条中的一个环节**

**关键实验要素**

1. **数据划分**
   - **错误做法**：随机打散全部序列（未来行为进入训练数据）
   - **正确做法**：按时间顺序划分（训练 → 验证 → 测试）
   - **其他泄漏来源**：物品热度统计、特征构造、负样本池

2. **负采样 vs 候选集**

| 比较维度 | 训练负采样           | 评价候选集                     |
|----------|----------------------|--------------------------------|
| 目的     | 构造模型学习样本     | 决定测试时从哪些物品中寻找目标 |
| 环节     | 模型训练             | 模型评价                       |
| 主要影响 | 模型学到的区分边界   | 指标难度和结果可比性           |

   - **不能混为一谈**：负样本数量、采样分布需要记录
   - **不能直接比较**：全量评价 vs 采样评价、不同候选集的Recall@K

3. **基线与公平比较**
   - **基线层次**：流行度 → ItemCF → 矩阵分解 → 合适的现代模型
   - **一致条件**：数据版本、预处理、划分、候选集、负采样、列表长度K、训练预算、调参条件
   - **不公平做法**：只选弱基线、不合理的基线参数、不同实验条件下比较

4. **评价指标**

| 指标       | 主要问题                         |
|------------|----------------------------------|
| Recall@K   | 相关物品中有多少被找回           |
| HitRate@K  | 推荐列表是否至少命中一个目标     |
| NDCG@K     | 命中物品是否位于列表前部         |
| RMSE/MAE   | 预测评分与真实评分相差多少       |

   - **准确性之外**：覆盖率、多样性、新颖性、用户满意度、效率、鲁棒性

5. **实验结论边界**
   - **可以说明**：模型在当前数据、任务、指标和设置下表现更好
   - **不能推广**：一个数据集代表所有领域、准确性提高代表多样性提高、离线指标代表线上体验

## 重点概念理解

### 1. 推荐系统的分类不是互斥的

一个方法可以同时具有多个标签：
- **频域序列推荐**：研究对象是序列，方法机制是频域
- **多模态图推荐**：研究对象是图，同时使用多模态信息
- **大模型增强的推荐模型**：大模型不是直接输出列表，而是增强特征

### 2. 召回、排序、重排是系统阶段，不是方法类别

- **错误理解**：把"召回模型"当作一类新算法
- **正确理解**：矩阵分解、协同过滤、生成式检索等方法都可以用于召回阶段
- **关键区别**：不同阶段对计算效率、精度、多样性的要求不同

### 3. 负采样的两面性

- **训练负采样**：构造学习样本，影响模型学到的边界
- **评价候选集**：决定测试难度，影响指标可比性
- **不能混淆**：同一个Recall@K，在不同候选集下无法直接比较

### 4. 时间信息泄漏是最常见的实验错误

**错误做法**：
- 随机打散序列 → 未来行为进入训练数据
- 使用未来统计量 → 用未来信息预测过去

**正确做法**：
- 按时间顺序划分
- 使用过去行为训练、较新行为验证、最后行为测试
- 所有特征和统计量只能使用当前时间点之前的数据

### 5. 公平比较的必要性

**不公平的例子**：
- 方法A用全量评价，方法B用采样评价
- 方法A推荐列表长度K=10，方法B的K=20
- 方法A训练100个epoch，方法B训练10个epoch
- 方法A使用未来信息，方法B严格按时间划分

**结果**：即使方法A指标更高，也无法证明方法本身更好

### 6. 频域和扩散的直觉理解

**频域推荐的价值**：
- 用户行为序列可能包含：周期性模式（每周五看电影）、随机噪声（误点击）、趋势变化
- 时域方法直接建模序列，频域方法分解为不同频率成分
- 类比：音乐的波形（时域） vs 频谱（频域）

**扩散推荐的价值**（我目前的理解）：
- 传统方法：直接拟合"用户点击了什么"
- 扩散方法：学习"用户可能喜欢什么的分布"
- 加噪-去噪过程可能让模型学会区分真实兴趣和噪声
- 多步去噪可能类似"粗到细"：先确定兴趣方向，再细化到具体物品

## 遇到的困惑

### 困惑1：扩散模型的加噪-去噪机制的深层原理

**我不理解的点**：
1. 为什么不直接学习用户偏好，而要先加噪再去噪？
2. 加噪的目的是数据增强、正则化，还是有更深层的原因？
3. 去噪网络学到的能力如何迁移到推荐任务？
4. 为什么要分成多步（50-100步）逐渐去噪，而不是一步到位？

**初步猜测**：
- 可能加噪-去噪让模型学习数据的生成分布，而不只是拟合观测
- 可能多步去噪在不同抽象层次上建模偏好（粗到细）
- 可能这个过程能更好地处理推荐数据中的稀疏性和噪声

**下一步**：精读DiffRec论文，看作者如何解释motivation

3. 课程提到"频率成分语义不直观"和"周期不稳定"，是否意味着频域方法只在特定场景有效？

## 学习感悟

1. **推荐系统的知识密度很大**：一节课涵盖了从经典到前沿、从方法到系统、从研究到合规的完整体系，需要反复消化

2. **方法不是孤立的**：同一个方法可以有多个标签，可以服务于多个系统阶段，关键是理解其本质和适用场景

5. **研究与工程的差距**：论文中的"一个模型"在真实系统中只是多路召回中的一路，需要理解完整的系统架构

6. **选择方向的策略**：频域和扩散都是2-3年的新方向，论文数量适中，既有研究空间又有一定基础，适合入门
